MSS智枢 · 能力架构全景与投资规划

企业内部 AI 员工操作系统（工作台）— 供业务沟通与阶段投资对齐 · 非对外销售材料

对内业务平台 营销服场景优先 投资分阶段推进
给业务领导的一句话： 员工用一句话说目标，平台匹配数字员工与能力，在任务中心完成计划确认与交付； 能力沉淀为 Agent/Skill，知识可检索溯源，多租户与权限支撑组织落地。

一、四层能力架构

L1 业务工作台

员工每天真正进入的入口

AI任务 任务中心 WarRoom 作战室
已可用 · 试点主路径

L2 数字员工与业务线

按事业部匹配相关 Agent

GTM / MKT 电商 / 零售 服务 / 渠道 / HR
已可用 · 场景待打穿

L3 能力资产中心

组织级可复用能力沉淀

Agent / Skill / Tool 知识库 RAG Prompt / Memory / Workflow
UI 齐全 · 深度不均

L4 治理与底座

对企业 IT / 安审可交代的部分

登录与成员角色 多租户 Runtime / API / LLM
演示级 → 生产级待投入

二、作业闭环（怎么用）

1说目标
→
2匹配数字员工
→
3确认计划
→
4执行对话
→
5交付物进会

关键步骤保留人审，避免「全自动替代」带来的责任悬空。

三、模块成熟度（投资对话）

层次 能力模块 业务价值 成熟度 下阶段投入重点
工作台AI任务降低使用门槛可用嵌入例会节奏
工作台任务中心 / WarRoom可协作、可交付可用交付物采纳与轻嵌入
业务线GTM～HR 数字员工场景匹配内容侧可用锁定 1 个金牌场景打穿
资产Agent / Skill能力复用可用绑定真数据与流程
资产知识库 RAG可溯源问答试点官方语料与权限分区
资产Workflow / 自动化例行任务雏形随场景需求再开
治理登录 / RBAC / 租户可管可控演示～有限生产审计与安审达标
底座Runtime / API / LLM可执行可扩展骨架+Mock生产 Runtime 与观测

四、三阶段投资规划

P0 · 4–8 周

阶段 1｜主路径打穿

收窄入口；1 个主办业务方场景；周活 + 交付物进例会。投入重心：业务运营 + 主路径，不堆新模块。

P1 · 1–2 季度

阶段 2｜可控扩展

轻嵌入企微/邮件/共享盘；准真数据；角色真实生效；复制到第 2 事业部/代表处。

P2 · 按安审与扩点

阶段 3｜平台加固

生产 Runtime、审计隔离、Workflow 编排、多场景水平扩展——此时再强化「OS」叙事。

五、扩点 / 预算前的三张「门票」

门票含义未达到时
主办业务方署名 有部门认领工序 先找试点业务方，再谈平台扩面
交付物被例会采纳 产出进入真实作业 停新模块，先把一个交付件跑进会
第二条复制路径 可复制，非一次性剧本 未复制前不按「全 MSS OS」立项

口径提醒：平台感是结果不是起点。对业务宜表述为「可试点的 AI 作业工作台 + 能力资产沉淀」； 「生产级治理 / 完整企业 OS」随阶段兑现后再升级说法。
