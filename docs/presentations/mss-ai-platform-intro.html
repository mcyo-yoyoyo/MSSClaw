01 · 平台定位

MSS AI提效作战平台

企业级 AI Employee Operating System —— 不是又一个聊天窗口， 而是把「说目标 → 做任务 → 出交付 → 沉淀能力」跑通的提效作战系统。

好学

逛广场找场景

AI广场按业务场景发现案例与工具；AI任务一句话开工。

好用

做任务出结果

计划确认、对话执行、交付物预览与推送，形成闭环。

可沉淀

学案例可复制

个人行动沉淀为组织资产：专家、技能、案例可复用。

一句话：让个人提效看得见，让组织能力用得上。

  • 怎么分层设计：业务壳 / 运营壳 + 能力平台
  • 能干什么：意图 → 计划 → 交付 → 推送
  • 谁怎么用：四类角色各走一条主路径

02 · 产品架构设计

双壳导航 × 能力平台 × 执行闭环

按角色自动进入业务工作台或平台运营台；复杂能力沉到运营侧，业务侧保持「逛 / 做 / 学」简洁。

体验层
逛广场 · AI任务 / AI广场 做任务 · 计划 / 对话 / 交付 学案例 我的消息 群聊（可配置）
能力层
专家 Agent 技能 Skill 工具 Tool 知识 RAG 记忆 Memory 提示词 / 工作流 / 自动化
治理层
展示配置（按角色菜单） 组织权限 租户 / 数据空间 门户运营
运行层
Plan 确认 执行流 / Mock·真实 LLM 交付物预览 推送作战室 / 成员

业务工作台

业务用户、只读访客默认进入。主路径：逛广场 → 做任务 → 学案例。不暴露专家配置与治理菜单。

平台运营台

超级管理员、能力运营默认进入。在业务路径之上，增加专家 / 技能发布与（超管）组织·租户·展示配置。

架构原则：导航保持短，能力按角色显隐 2 / 4
用「四层」讲：体验给一线用，能力给运营建，治理给超管控，运行层把任务真正跑起来。 强调双壳不是两套产品，而是同一平台按角色裁剪后的两种工作面。

03 · 核心功能特性

一条提效闭环，打穿日常工作

从发现到交付，尽量少跳转；侧栏只保留「最近在办」，避免会话列表变成聊天垃圾桶。

01
逛 · 发现与开工 AI广场找场景/工具；AI任务描述目标、@专家、/技能
→
02
做 · 计划与执行 自动提炼短任务名；Plan 确认后执行；状态可读
→
03
交 · 预览与推送 交付物面板预览；推送到作战室或「我的消息」
→
04
学 · 复制与沉淀 案例样板间一键复用；专家/技能持续上架

场景驱动，而非工具堆叠

  • 广场按能力/场景分类引导
  • 案例可复制进任务
  • 专家团可同会话接力

任务体验轻量可控

  • 侧栏默认最近活跃 ≤8
  • 规则 + AI 生成短标题
  • 双击可手动重命名

权限与展示可治理

  • 四角色 RBAC 矩阵
  • 按角色配置侧栏菜单
  • 只读访客不可执行
核心承诺：说得出 → 做得完 → 推得出 → 学得会 3 / 4
这一页用闭环串功能，避免变成功能清单。若听众追问技术，可补充：前端 React 工作台， 可选接 API/LLM；无 Key 时 Mock 演示，有 Key 时走真实模型。

04 · 四类用户视角

同一平台，四条清晰主路径

登录即按角色进入对应壳层；菜单与权限一致，避免「看得见却点不动」。

业务工作台

演示建议： 先用业务用户走通「AI任务 → 做任务 → 推送消息」； 再切换能力运营展示「专家/技能上架」； 最后用超管打开「展示配置」说明菜单如何按角色裁剪。
